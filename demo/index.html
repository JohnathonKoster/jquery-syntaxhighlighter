About

jQuery Syntax Highlighter provides an elegant and simplistic interface for using version of Alex Gorbatchev's Syntax Highlighter with jQuery. We also add a series of features that aren't possible just using Alex's Syntax Highlighter, to ensure that you get the best experience possible:

  • You can now install and initialise with only a few lines of code instead of the whole bunch you had to include before.
  • Support for wrapLines is back in action!
  • Support for using code elements for SEO and semantic relevance!
  • Support for the HTML5 data attribute data-sh allowing you to no longer have to pollute your CSS classnames when you want to Syntax Highlight! This follows the WhatWG's recommendation for include code into your website (they make HTML5). Note: We still support the old way of polluting the CSS classnames though for backwards compatibility.

So let's walk through it.

Installation

Step 1. Download jQuery Syntax Highlighter, and extract it to your hard drive

Step 2. Move the extracted jquery-syntaxhighlighter directory to somewhere on your webserver

If all the files were extracted without being in a directory, create a new directory called jquery-syntaxhighlighter and place the extracted files in that.

Step 3. Include jQuery (insert into your page's head tag)

If your page already has jQuery included then you can skip this step.

<!-- Include jQuery (Syntax Highlighter Requirement) --> <script type="text/javascript" src="http://www.yoursite.com/somewhere/on/your/webserver/jquery-syntaxhighlighter/scripts/jquery-1.4.2.min.js"></script>

Step 4. Include jQuery Syntax Highlighter (insert into your page's head tag)

<!-- Include jQuery Syntax Highlighter --> <script type="text/javascript" src="http://www.yoursite.com/somewhere/on/your/webserver/jquery-syntaxhighlighter/scripts/jquery.syntaxhighlighter.min.js"></script>

Step 5. Initialise jQuery Syntax Highlighter (insert into your page's head tag)

<!-- Initialise jQuery Syntax Highlighter --> <script type="text/javascript">$.SyntaxHighlighter.init();</script>

And that's all there is to it in regards to installation. So how do we let Syntax Highlighter that we want to highlight our code block?

Usage

Using the WhatWG's recommendation:

So you have you standard code block of let's say javascript code, which looks like this:

This is the code block that I would to syntax highlight: <code> // I am a JavaScript code block and it is my aspiration in life to be highlighted. (function(aspirationInLife){ return (aspirationInLife === 'SyntaxHighlighted') ? false : 'Unknown aspiration!?!'; })('SyntaxHighlighted'); </code>

Great, so now let's just add the classname which the WhatWG recommends which is language-javascript to our code element. Let's see what that looks like:

This is the code block that I would to syntax highlight: <code class="language-html"> // I am a JavaScript code block and it is my aspiration in life to be highlighted. (function(aspirationInLife){ return (aspirationInLife === 'SyntaxHighlighted') ? true : 'Unknown aspiration!?!'; })('SyntaxHighlighted'); </code>

So that's pretty nifty isn't it! Now let's say we want to remove the gutter (the line numbers), how can we do that?

Removing the Gutter

We can do this two ways: The first is globally so it applies to all syntax highlight instances. The second is locally so it applies to only the instances we set it to.

So let's see how we would do this globally. We would go back to our page's head element to where we initialise the Syntax Highlighter plugin. Once we are there, we would add some configuration so it would like this:

<!-- Initialise jQuery Syntax Highlighter --> <script type="text/javascript"> $.SyntaxHighlighter.init({ 'defaults': { 'gutter':false } }); </script>

Sweet, so now all the syntax highlighters on our page won't use the gutter. But what if we would only like to apply this per instance. Thanks to the jQuery Syntax Highlighter extension we can do this via the HTML5 data attribute data-sh, so let's see how that would work:

This is the code block that I would to syntax highlight: <code class="language-html" data-sh='{"gutter":false}'> // I am a JavaScript code block and it is my aspiration in life to be highlighted. (function(aspirationInLife){ return (aspirationInLife === 'SyntaxHighlighted') ? true : 'Unknown aspiration!?!'; })('SyntaxHighlighted'); </code>

Now isn't that sweet! For a full list of defaults (options that work per instance), and configs (options that work globally) you can check out the Syntax Highlighter documentation here. There is one config option that is not covered in that link, as it only exists in the jQuery Syntax Highlighter extension, and that is the ability to wrap lines inside the syntax highlighters.

Wrapping Lines

Wrapping lines is very important if you have very long pieces of code, as otherwise if the user does not have a touch device, then if the code is not wrapped it is no obvious that they can scroll to it as the scroll bar is out of the current visible far below at the end of the code, so they would have to scroll down, move the scroll bar, scroll up, and try and find out where on earth they left off. It's just soo confusing and not nice if you are using long lines of code. Thankfully we have the wrap lines feature in our jQuery Syntax Highlighter extension, so here is how we use it.

All we have to do is set the wrapLines config option to true, we do this by going back up the our page's head element where we initialise our Syntax Highlighter and change it to:

<!-- Initialise jQuery Syntax Highlighter --> <script type="text/javascript"> $.SyntaxHighlighter.init({ 'config': { 'wrapLines':true } }); </script>

Alternatively if we want to wrap lines AND remove the gutter globally we can do this by merging the configuration. Let's see what that looks like:

<!-- Initialise jQuery Syntax Highlighter --> <script type="text/javascript"> $.SyntaxHighlighter.init({ 'defaults': { 'gutter':false }, 'config': { 'wrapLines':true } }); </script>

Well that's it, it really is that simple. Good luck and hope you like it. If you ever need help or support you can send feedback by the button on the right or the "Get Support" link up the top. If you want to get cracking the download link is up the top with the rest of the links.

Encoding HTML

One extremely important thing is encoding your HTML properly when you use them in code samples. This is because if you don't your web browser will think it has to run that code, despite it being in a code element! Argh! That could really cause problems... So what we want to do is whenever we want to include code blocks (regardless of whether or not we are wanting to syntax highlight them) we always must remember to encode the HTML entities appropriately! So we do this by converting all the characters which are < to &lt; and all the > to &gt;. So for instance let's look at the following code sample:

<code class="language-html"> <span> I'm a span element. I hope I've been escaped properly... as otherwise I will be :-( and the browser will be :-O </span> </code>

Looks good, but this is how we actually code it in our HTML:

<code class="language-html"> &lt;span&gt; I'm a span element. I hope I've been escaped properly... as otherwise I will be :-( and the browser will be :-O &lt;/span&gt; </code>

Hrmmm. Interesting... But once you've got that nailed, then there really are not worries and you're all good to go. Alternatively you could look at the source code for this demo to gain a better grasp of this issue.
